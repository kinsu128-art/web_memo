# 메모관리 프로그램 PRD (Product Requirements Document)

## 1. 프로젝트 개요

**프로젝트명:** 메모관리 웹 서비스
**목표:** 사용자가 웹 인터페이스를 통해 메모를 작성, 수정, 삭제, 조회할 수 있는 간단한 메모관리 애플리케이션 제공

---

## 2. 기술 스택

- **프론트엔드:** HTML, CSS, JavaScript (또는 React/Vue 등)
- **백엔드:** Node.js/Python/Java 등
- **데이터베이스:** MySQL 8.0+
- **배포 환경:** Docker & Docker Compose
- **웹 서버:** Nginx (리버스 프록시) 또는 내장 서버

---

## 3. 주요 기능 요구사항

### 3.1 메모 관리 기능
- **메모 작성**: 제목과 내용을 입력하여 새 메모 생성
- **메모 조회**: 작성된 모든 메모 목록 표시
- **메모 상세 조회**: 개별 메모의 전체 내용 확인
- **메모 수정**: 기존 메모의 제목 및 내용 편집
- **메모 삭제**: 메모 삭제

### 3.2 메모 정보
- **제목** (필수): 메모의 제목
- **내용** (필수): 메모의 본문
- **생성일시**: 메모 생성 시간 (자동 기록)
- **수정일시**: 마지막 수정 시간 (자동 기록)

### 3.3 사용자 인터페이스
- **메인 페이지**: 메모 목록 및 생성 폼
- **메모 상세 페이지**: 메모 내용 표시 및 수정/삭제 버튼
- **반응형 디자인**: 모바일 및 데스크톱 호환

---

## 4. 데이터베이스 설계

### 4.1 메모 테이블 (memos)
```
- id (INT, PRIMARY KEY, AUTO_INCREMENT)
- title (VARCHAR(255), NOT NULL)
- content (LONGTEXT, NOT NULL)
- created_at (TIMESTAMP, DEFAULT CURRENT_TIMESTAMP)
- updated_at (TIMESTAMP, DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP)
```

## 7. API 명세 (백엔드 기준)

### 7.1 메모 목록 조회
- **Endpoint**: `GET /api/memos`
- **응답**: 메모 목록 (JSON)

### 7.2 메모 상세 조회
- **Endpoint**: `GET /api/memos/:id`
- **응답**: 메모 상세 정보 (JSON)

### 7.3 메모 생성
- **Endpoint**: `POST /api/memos`
- **요청**: `{ title, content }`
- **응답**: 생성된 메모 (JSON)

### 7.4 메모 수정
- **Endpoint**: `PUT /api/memos/:id`
- **요청**: `{ title, content }`
- **응답**: 수정된 메모 (JSON)

### 7.5 메모 삭제
- **Endpoint**: `DELETE /api/memos/:id`
- **응답**: 삭제 성공 메시지
